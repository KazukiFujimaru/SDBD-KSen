Tipe Data Abstrak

Dalam pemrograman, struktur data merupakan cara untuk mengorganisir dan menyimpan data agar dapat diakses dan dimodifikasi secara efisien. Beberapa struktur data telah didefinisikan secara langsung oleh bahasa pemrograman, seperti integer, string, dan array. Namun, ada juga struktur data yang tidak tersedia secara langsung dan harus didefinisikan oleh programmer sendiri. Struktur data yang didefinisikan oleh programmer ini disebut sebagai Abstract Data Type (ADT). ADT adalah konsep yang mendefinisikan cara kerja dan perilaku suatu struktur data tanpa menentukan implementasi spesifiknya.


Tipe Data Abstrak

Tipe Data Abstrak (ADT) adalah konsep model untuk struktur data yang mendefinisikan operasi-operasi yang dapat dilakukan pada data tanpa menentukan bagaimana operasi tersebut diimplementasikan. ADT memberikan kerangka struktur untuk apa yang data bisa lakukan dan apa yang data tidak bisa lakukan.

Karena ADT hanya berfokus pada logika operasi dan tidak bergantung pada cara kerja bahasa pemrograman, konsep ini dapat diterapkan di berbagai bahasa pemrograman. Implementasi ADT dibuat secara langsung dalam kode oleh pengembang, karena bahasa pemrograman umumnya tidak menyediakan ADT secara langsung dan hanya menyediakan alat atau tipe data dasar untuk membangunnya.


Penerapan ADT

Penerapan ADT dalam bahasa pemrograman dilakukan langsung di dalam kode dengan menggunakan fungsi. Fungsi yang telah dibuat ini nantinya akan digunakan dalam kode untuk menjalankan ADT.


Fungsi (Function) dalam pemrograman

Fungsi dalam pemrograman adalah sekumpulan instruksi yang dapat dijalankan secara berulang-ulang. Fungsi dapat menerima argumen (data yang dikirim ke fungsi dan diterima oleh fungsi) dan mengembalkan nilai (mengirim kembali hasil dari proses yang dilakukan oleh fungsi). Fungsi dapat digunakan untuk memecah kode dan mengelompoklkan kode menjadi bagian-bagan yang lebih kecil dan lebih mudah dipahami.


Jika disimpulkan, esensi Tipe Data Abstrak adalah ?

Secara umum, konsep ADT diterapkan dengan mendefinisikan operasi yang bisa dilakukan pada data. Dalam kode, ADT umumnya dibuat menggunakan sekumpulan fungsi.

Meskipun sering diimplementasikan dengan fungsi, inti dari ADT adalah pemisahan antara spesifikasi dan implementasi.

Spesifikasi yang dimaksud adalah menentukan apa yang bisa dilakukan, seperti menambah atau menghapus data, tanpa menjelaskan cara melakukannya.

Sementara itu, implementasi adalah bagaimana spesifikasi tersebut dijalankan dalam kode, termasuk bagaimana data disimpan dan diproses.

Jenis-jenis ADT

Berikut adalah jenis-jenis Tipe Data Abstrak yang dijelaskan secara sederhana :

Array

Array adalah kumpulan data dengan ukuran tetap yang bisa diakses dengan indeks. Array bisa diibaratkan seperti rak buku dengan nomor urut. Setiap buku (data) bisa diambil dengan melihat nomornya (indeks).

Contoh penerapannya adalah : Daftar nilai ujian siswa [80, 90, 75, 88, 95]. Disini Siswa berukuran tetap tanpa penambahan atau pengurangan secara langsung.
Operasi utama :
  • Menyimpan data di indeks tertentu set(index, value)
  • Mengambil data dari indeks get(index)
  • Mengetahui ukuran array size()


List (Daftar)

List adalah kumpulan data yang bisa bertambah atau berkurang ukurannya (secara dinamis). List bisa diibaratkan seperti daftar belanja yang bisa diisi atau dihapus kapan saja.

Contoh penerapannya adalah ["Apel", "Pisang", "Jeruk"]. Disini data belanjaan bisa ditambah atau dihapus dengan mudah.
Operasi utama :
  • Menambahkan elemen append()
  • Menghapus elemen remove()
  • Mencari elemen search()


Stack (Tumpukan)

Struktur data dengan konsep "Last In, First Out" (LIFO), yaitu dimana data yang terakhir masuk adalah data yang pertama keluar. Stack dapat diibaratkan seperti tumpukan piring di restoran. Piring terakhir yang ditumpuk harus diambil dulu sebelum mengambil yang lain.

Contoh penerapannya adalah fitur Undo/Redo. Fitur Undo/Redo menggunakan aksi terakhir yang kita gunakan (yang berarti data yang terakhir masuk).
Operasi utama :
  • Push, yaitu Menambahkan data ke tumpukan
  • Pop, yaitu Mengambil data dari tumpukan
  • Peek, yaitu Melihat data teratas tanpa mengambilnya


Queue (Antrian)

Queue, adalah struktur data dengan konsep "First In, First Out" (FIFO), yaitu dimana data yang pertama masuk adalah data yang pertama keluar. Queue dapat diibaratkan seperti antrian di kasir. Orang pertama yang datang akan dilayani dulu.

Contoh penerapannya adalah antrian pencetakan dokumen pada printer. Dokumen yang pertama kali dicetak adalah dokumen yang pertama kali tugaskan untuk print.
Operasi utama :
  • Enqueue, yaitu menambahkan data ke antrian
  • Dequeue, yaitu mengambil data dari antrian


Linked List (Daftar Berantai)

Linked List adalah struktur data yang terdiri dari elemen-elemen yang saling terhubung. Linked List dapat diibaratkan seperti rantai sepeda, di mana setiap mata rantai terhubung ke yang berikutnya.

Contoh penerapannya adalah navigasi lagu dalam playlist musik. Playlist musik dapat berubah urutan secara langsung ketika dimainkan walaupun sudah disusun sebelumnya. Linked list ini khususnya tercermin di fitur Shuffle/Lagu Acak dimana urutan lagu tetap sama namun urutan putar berbeda.
Operasi utama :
  • Menambahkan elemen insert()
  • Menghapus elemen delete()
  • Menelusuri elemen traverse()


Graph (Graf)

Graf adalah struktur data yang berisi kumpulan titik (node) yang saling terhubung. Graf dapat diibaratkan seperti peta jalan, di mana kota-kota adalah node dan jalan adalah hubungan antar node.

Contoh penerapannya adalah jaringan sosial pada media sosial. Media sosial menggunakan algoritma rekomendasi konten dan teman berdasarkan hubungan pengguna.

Operasi utama :
  • Menambah simpul addNode()
  • Menambah hubungan addEdge()
  • Mencari jalur findPath()


Tree (Pohon)

Pohon adalah struktur data hierarkis di mana setiap elemen memiliki hubungan "anak" dan "induk". Pohon dapat diibaratkan seperti pohon keluarga, dimana ada orang tua dan anak-anaknya. Selain itu bisa diibaratkan juga dengan struktur organisasi dan organigram, dimana ada ketua(induk) dan pengurus-pengurus lainnya (anak).

Contoh penerapannya adalah struktur folder di komputer. Pada struktur folder di komputer, satu folder bisa memiliki turunan yang merupakan isi dari folder. Isi folder ini dapat berupa file (anak) atau berupa folder lain (induk lain)
Operasi utama :
  • Menambahkan node insert()
  • Mencari node search()
  • Menghapus node delete()


Contoh Implementasi ADT

Implementasi ADT dilakukan dengan menerapkan aturan-aturan ADT pada program yang kita miliki. Disini kita akan menerapkan konsep Array pada bahasa pemrograman Python.
Sebelum lanjut, mari kita pahami dulu bahwa sifat-sifat array adalah :

  • Ukurannya tetap.
  • Elemen disimpan secara berurutan.
  • Elemen dapat diakses dengan index.
  • Tipe Data Homogen (semua tipe data pada Array sama).

Dengan sifat-sifat tersebut, maka kita dapat membuat array dengan kode berikut :

class ArrayADT:
def __init__(self, ukuran, tipe_data):
    # Membuat array dengan ukuran tetap dan tipe data tertentu.
    self.isi = [None] * ukuran  # Array dengan ukuran tetap
    self.panjang = ukuran
    self.tipe_data = tipe_data  # Menyimpan tipe data yang diperbolehkan

def set(self, index, value):
    # Menetapkan nilai pada indeks tertentu, dengan validasi tipe data.
    if 0 <= index < self.panjang:
        if isinstance(value, self.tipe_data):
            self.isi[index] = value
            print(f"Set isi[{index}] = {value}")
        else:
            print(f"Error: Nilai harus bertipe {self.tipe_data.__name__}!")
    else:
        print("Error: Indeks di luar batas!")

def get(self, index):
    # Mengambil nilai dari indeks tertentu, jika indeks valid.
    if 0 <= index < self.panjang:
        return self.isi[index]
    else:
        print("Error: Indeks di luar batas!")
        return None

def ukuran(self):
    # Mengembalikan ukuran array.
    return self.panjang

def display(self):
    """Menampilkan isi array."""
    print(self.isi)

# Contoh penggunaan
# Membuat array integer dengan ukuran 5
arr = ArrayADT(5, int)  

# Berhasil karena sesuai tipe data
arr.set(0, 10)
arr.set(3, 20)  
arr.display()

# Gagal karena tidak sesuai tipe data
arr.set(3, "Ini String")  

# Gagal karena indeks di luar batas
arr.set(5, 50)  

# Menampilkan satu data
print("Nilai di indeks 3:", arr.get(3))

# Menampilkan panjang array
print("Ukuran array:", arr.ukuran())

Kode ini adalah implementasi sederhana dari konsep Array sebagai Abstract Data Type (ADT) dalam Python. Array adalah sekumpulan data yang disusun secara berurutan dan diakses menggunakan indeks. Dalam kode ini, dibuat sebuah kelas bernama ArrayADT yang berfungsi sebagai wadah untuk menyimpan data dengan aturan tertentu. Saat kelas array dibuat, kita sudah menuliskan kode untuk menentukan ukuran, dan tipe data dari array.

Setiap elemen dalam array disimpan dalam daftar (list) dengan ukuran tetap, di mana semua slot awalnya berisi None sebagai tanda bahwa belum ada data yang dimasukkan. Untuk memasukkan nilai ke dalam array, kita membuat fungsi set() yang memastikan bahwa nilai yang diberikan sesuai dengan tipe data yang telah ditentukan dan indeks yang diberikan tidak diluar batas. Jika pengecekan berhasil, data akan dimasukkan ke Array dan jika pengecekan gagal, pesan error akan muncul.

Untuk mengambil data dari array, tersedia fungsi get(), yang mengembalikan nilai dari indeks tertentu jika masih didalam batas ukuran. Selain itu, ada juga fungsi ukuran() yang mengembalikan nilai jumlah elemen maksimum yang dapat disimpan dalam array. Terakhir, fungsi display() digunakan untuk menampilkan seluruh isi array saat ini.

Kode ini menunjukkan bagaimana ADT bekerja dalam pemrograman, di mana aturan-aturan tertentu diterapkan untuk mengelola data dengan lebih rapi dan terkendali.

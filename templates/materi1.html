Pengantar Struktur Data dan Basis Data

Komputer pada dasarnya tidak pintar, kita membuat komputer jadi pintar. Kita semua adalah ahli sihir yang memasukan simbol-simbol aneh (Bahasa Pemrograman) untuk memaksa seperangkat batu-batu ajaib yang memiliki ukiran khusus (Perangkat Keras Komputer) untuk dapat melakukan hal-hal ajaib. Dengan ilmu yang awalnya dikuasai oleh sesepuh luhur(Ilmuwan Informatika), komputer menjadi lebih bisa diakses oleh semua kalangan dengan menurunkan jurus yang lebih sederhana(Abstraksi pada Komputer). Akhirnya, ilmu ini bahkan dapat digunakan dan dikuasai secara bertahap oleh teman-teman semua.


Esensi Kerja Komputer

Komputer hanya memahami bahasa mesin, yang sering disebut dengan bahasa biner. Bahasa biner ini kemudian dikonversi menjadi bentuk yang lebih mudah dipahami oleh manusia (seperti huruf). Dari huruf-huruf ini terbentuk bahasa, dan dari bahasa terbentuk bahasa pemrograman.

Bahasa pemrograman menyediakan metode yang mudah agar kita dapat membuat suatu program pada komputer. Namun, dengan begini kebutuhan pemrograman pun semakin kompleks, dan jumlah data yang diproses makin besar. Oleh karena itu, diperlukan struktur yang jelas dan efisien dalam pengelolaan data agar sistem dapat bekerja secara optimal.


Struktur Data

Struktur data adalah cara mengorganisir dan menyimpan data dalam komputer sehingga dapat digunakan secara efisien. Struktur data mencakup berbagai metode penyimpanan data, seperti array, linked list, stack, queue, tree, dan graph. Masing-masing struktur data memiliki karakteristik dan kegunaannya sendiri tergantung pada kebutuhan aplikasi.


Mengapa Struktur Data Diperlukan?

Struktur data sangat penting dalam pengembangan perangkat lunak karena berkaitan erat dengan efisiensi algoritma. Dua aspek utama yang dipertimbangkan dalam pemilihan struktur data adalah:

  • Time Complexity: Waktu yang dibutuhkan untuk menjalankan suatu operasi pada struktur data. Misalnya pada pencarian data.
  • Space Complexity: Jumlah memori yang digunakan oleh struktur data. Misalnya pada besarnya RAM yang dipakai dalam jalannya program.


Apa esensi dari Struktur Data ?

Sesuai dengan tujuannya, struktur data mengoptimalkan penggunaan waktu (time complexity) dan ruang (space complexity) saat data diproses oleh program.

Walaupun konsep struktur data sekilas terlihat rumit, namun struktur data ini membantu para programmer untuk membuat aplikasi yang lebih ringan dan reponsif.

Namun struktur data ini bukan untuk visualisasi data. Struktur data sendiri lebih berfokus pada penyimpanan dan pengolahan data di balik layar.

Contoh Penerapan Sederhana Struktur Data

Sebagai contoh sederhana, kita bisa melihat bagaimana struktur data stack digunakan dalam fungsi undo/redo. Stack bekerja dengan prinsip Last In, First Out (LIFO), di mana elemen terakhir yang dimasukkan adalah elemen pertama yang diambil. Berikut adalah contoh implementasi stack dalam Python: 

stack = []  # Membuat array stack
stack.append('A')  # Menambahkan nilai A ke array
stack.append('B')  # Menamnbahkan nilai B ke array
stack.append('C')  # Menambahkan nilai C ke array
print('x') # Menampilkan semua isi array
print(stack.pop())  # Output: C

'stack' disana adalah variabel dengan tipe array. Array dapat berisi sekumpulan elemen yang berbeda walaupun memiliki jenis data yang berbeda. Dalam contoh di atas, array stack berisi tiga elemen, yaitu A B dan C.

Karena 'stack' pada kode tersebut adalah variabel, maka kata 'stack' bisa diubah sesuai dengan keinginan anda.

Adapun untuk fungsi yang digunakan pada kode ini adalah :
(variabel).append = menambah elemen di urutan paling akhir array.
(variabel).pop = mengeluarkan elemen di urutan paling akhir array.

Untuk belajar lebih lanjut, teman-teman dapat mencoba kode Python tersebut di aplikasi yang teman-teman sudah pasang (seperti Visual Studio Code atau IDE Python) atau di website yang dapat menjalankan kode python (seperti Programiz, OnlinePython, dan program lain yang serupa)

Basis Data

Basis Data atau yang sering disebut Database adalah kumpulan data dengan tipe-tipe berbeda yang terorganisir dan disimpan dalam bentuk digital serta dapat diakses secara sistematis (biasanya melalui DBMS). Database dapat berisi huruf, nomor, foto, video, dokumen, dan data-data lainnya.

Jika teman-teman telusuri, Database sendiri memiliki berbagai macam database sesuai dengan kebutuhan aplikasinya. Namun, secara luas; database dapat dibagi menjadi dua, yaitu database dengan sistem tabel (SQL Database) dan database dengan sistem non tabel (NoSQL Database).

Adapun pada praktikum ini, kita akan menggunakan bahasa SQL karena bahasa SQL memudahkan pemahaman tentang bagaimana Database bekerja.


Bagaimana Database Digunakan ?

Dalam suatu aplikasi, database digunakan untuk menyimpan, mengelola, dan mengambil data agar aplikasi dapat berjalan. Penggunaan database dapat ditemukan dalam aksi-aksi seperti:

  • Pendaftaran dan login pengguna ke aplikasi, aplikasi akan menggunakan data username dan password pengguna di database.
  • Upload foto ke sosial media, aplikasi akan menerima foto yang akan di upload dan menyimpan foto tersebut ke database.
  • Pembayaran barang di aplikasi E-Commerce, barang yang sudah dibayar akan diubah statusnya di database agar hilang dari keranjang dan masuk ke status pengiriman.


Manakah pernyataan berikut yang benar tentang database?

Database hanya berhubungan dengan hal-hal yang berkaitan dengan pengelolaan data pada aplikasi.
Memang database dapat merubah tampilan dengan menghilangkan data, membantu keamanan aplikasi, dan membuat fungsi aplikasi yang berkaitan dengan data dapat berjalan dengan sesuai. Tetapi aspek-aspek tersebut dapat dilakukan dengan kolaborasi antar bagian dari aplikasi.

Contoh Penerapan Sederhana Basis Data

Sebagai contoh sederhana, kita bisa melihat bagaimana data disimpan dalam database menggunakan SQL. Berikut adalah contoh pembuatan tabel dan penyisipan data dalam database menggunakan SQL: 

-- Pembuatan tabel 'Mahasiswa'.
CREATE TABLE Mahasiswa (  
ID INT PRIMARY KEY,  
Nama VARCHAR(100),  
Jurusan VARCHAR(50)  
); 
-- Memasukan data kedalam tabel 'Mahasiswa'.
INSERT INTO Mahasiswa (ID, Nama, Jurusan) VALUES (1, 'Lysander', 'Teknologi Informasi');
-- Mengambil data dar tabel 'Mahasiswa' dan menampilkannya.
SELECT * FROM Mahasiswa;

'Mahasiswa' di sini adalah nama tabel yang akan dibuat dalam database. Tabel ini berfungsi untuk menyimpan data mahasiswa dengan beberapa atribut, yaitu ID, Nama, dan Jurusan.

Pada database, kita dapat membuat tabel sesuai dengan keperluan dan dapat menyesuaikan struktur dari tabel tersebut. Misalnya jika teman-teman ingin menambah kolom baru dengan atribut tambahan. Selain itu, nama tabel itu bebas dan dapat disesuaikan dengan kebutuhan.

Adapun untuk perintah yang digunakan pada kode ini adalah:
CREATE TABLE (nama_tabel) (...); - Digunakan untuk membuat tabel baru dalam database.
INSERT INTO (nama_tabel) VALUES (...); - Digunakan untuk menambahkan data ke dalam tabel yang telah dibuat.
SELECT * FROM (nama_tabel); -Digunakan untuk mengambil dan menampilkan semua data dari tabel yang dipilih.

Untuk belajar lebih lanjut, teman-teman dapat mencoba kode SQL tersebut di aplikasi DBMS (seperti SQLyog, DB Browser, dan aplikasi serupa) atau di website yang dapat menjalankan kode SQL (seperti Programiz, SQliteOnline, dan program lain yang serupa)

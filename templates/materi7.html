Manipulasi Data & Transaksi

Apa itu Manipulasi Data?

DBMS pasti memiliki fitur untuk mengelola data, sehingga DBMS pasti memiliki fitur untuk memanipulasi data. Manipulasi data mencakup operasi Create, Read, Update, dan Delete yang dikenal dengan istilah CRUD. SQL sebagai bahasa standar untuk mengakses dan memanipulasi database menyediakan sintaks lengkap untuk operasi manipulasi data. Tanpa kemampuan ini, database hanya menjadi tempat penyimpanan statis yang tidak bisa digunakan secara dinamis oleh aplikasi.


Manipulasi data adalah proses perubahan data yang sudah ada dalam sistem. Dalam database, manipulasi data berarti melakukan operasi terhadap isi tabel menambahkan data baru, membaca data yang sudah ada, mengubah data, atau menghapusnya. Operasi ini memungkinkan kita menjaga informasi dalam database agar tetap relevan dan akurat terhadap kebutuhan aplikasi atau bisnis.

Lebih dari sekadar CRUD, SQL juga memiliki cara untuk manipulasi yang lebih kompleks melalui fitur seperti WHERE untuk menyaring data, ORDER BY untuk mengurutkan hasil, dan LIMIT untuk membatasi jumlah data yang ditampilkan. Fitur-fitur ini memperkuat kontrol pengguna terhadap data yang sedang dimanipulasi, serta meningkatkan efisiensi dan fleksibilitas pengolahan data.


Create (INSERT): Menambahkan data baru ke dalam tabel

INSERT INTO mahasiswa (nama, jurusan) VALUES ('John Doe', 'Informatika');

Read (SELECT): Membaca/mengambil data dari tabel

SELECT * FROM mahasiswa WHERE jurusan = 'Informatika';

Update (UPDATE): Mengubah data yang sudah ada

UPDATE mahasiswa SET nilai = 85 WHERE nama = 'John Doe';

Delete (DELETE): Menghapus data dari tabel

DELETE FROM mahasiswa WHERE nama = 'John Doe';

Transaksi Database

Saat memanipulasi data, sangat penting untuk memastikan bahwa proses berjalan dengan aman dan tidak merusak integritas data. Kesalahan sekecil apapun dalam operasi seperti penghapusan atau pengubahan bisa berdampak besar, apalagi jika dilakukan secara bersamaan oleh banyak pengguna. Untuk mengatasi ini, sistem database menyediakan fitur transaksi.

Transaksi adalah serangkaian operasi database yang harus dieksekusi sebagai satu kesatuan. Jika satu bagian dari transaksi gagal, maka seluruh operasi bisa dibatalkan untuk menghindari kerusakan data. Transaksi harus mematuhi konsep ACID: Atomicity, Consistency, Isolation, dan Durability (pelajari lebih lanjut di sini). Istilah "transaksi" sendiri diambil dari dunia keuangan, menggambarkan suatu pertukaran yang harus diproses secara utuh — semua atau tidak sama sekali.


Atomicity: Transaksi harus berjalan secara utuh - semua operasi berhasil atau semuanya dibatalkan.

Consistency: Database harus tetap dalam keadaan yang valid sebelum dan sesudah transaksi.

Isolation: Transaksi yang berjalan bersamaan tidak saling mengganggu.

Durability: Hasil transaksi yang berhasil harus tetap tersimpan meskipun sistem mengalami kegagalan.


Mengapa Manipulasi Data Dijalankan pada Query?

Beberapa fungsi manipulasi data seperti penyaringan (WHERE), pengurutan (ORDER BY), dan pembatasan hasil (LIMIT) sebenarnya bisa juga dilakukan di sisi aplikasi setelah data diambil dari database. Fakta ini mungkin terlihat fleksibel, terutama dalam kasus sederhana.

Namun, menjalankan manipulasi ini langsung di query SQL jauh lebih efisien karena beban kerja diserahkan ke server database yang memang dioptimalkan untuk melakukan operasi tersebut. Manipulasi yang dilakukan oleh query pada database mengurangi beban pada aplikasi dan menghemat bandwidth (jumlah maksimal pengiriman yang bisa dilakukan pada saat proses pertukaran data) karena data yang diambil sudah tersaring sejak awal.


Kapan Menggunakan Query vs Aplikasi?

Secara umum, manipulasi data sebaiknya dilakukan di query ketika:

  • Data yang ditarik sangat besar
  • Proses manipulasi bisa diselesaikan oleh SQL
  • Performa sistem terbatas atau diprioritaskan

Sebaliknya, manipulasi oleh aplikasi bisa berguna saat logika terlalu kompleks untuk SQL atau perlu dikombinasikan dengan input pengguna secara langsung. Memahami kapan menggunakan query dan kapan menggunakan kode aplikasi adalah bagian penting dari desain sistem yang efisien.


-- Contoh query dengan manipulasi data
SELECT nama, nilai 
FROM mahasiswa 
WHERE jurusan = 'Informatika' AND nilai > 80 
ORDER BY nilai DESC 
LIMIT 10;

Jadi, mengapa manipulasi data sebaiknya dilakukan di query database daripada di aplikasi?

Manipulasi data di query database berarti query dijalankan pada server database sehingga mengurangi beban pada aplikasi ketika berjalan. Karena data sudah proses, berarti data yang diambil dari database tidak banyak dan data yang diambil tidak perlu diproses ulang sehingga akhirnya beban pada aplikasi berkurang.

Pengantar Basis Data (Database)

Bagaimana Komputer Menyimpan Data

Komputer menyimpan data dalam bentuk bit dan byte di media penyimpanan seperti harddisk, SSD, atau memori. Untuk memudahkan pengguna, data ini dikemas dalam struktur file yang bisa dikenali oleh sistem operasi, misalnya .txt, .docx, .jpg, dan lainnya. Setiap file berisi informasi tertentu dan bisa dibaca oleh aplikasi yang sesuai. Misalnya, file .mp3 dibaca oleh pemutar musik, dan file .json dibaca oleh aplikasi yang mengelola data.

Namun, metode penyimpanan ini memiliki keterbatasan ketika data menjadi kompleks, saling berkaitan, dan perlu diakses oleh banyak pengguna secara bersamaan. Di sinilah konsep database muncul sebagai solusi.


Kelebihan dan Keterbatasan File Biasa

Kelebihan:
  • Mudah dibuat dan digunakan (misal .txt, .csv)
  • Tidak memerlukan perangkat lunak tambahan
  • Cocok untuk data sederhana, jumlah kecil
Keterbatasan:
  • Tidak efisien untuk mencari atau memfilter data besar
  • Sulit untuk menjaga integritas dan konsistensi
  • Tidak mendukung relasi antar data
  • Tidak aman untuk akses banyak pengguna sekaligus

Mengapa Database Ada?

Secara umum, penggunaan database memberikan:

  • Efisiensi: Database dapat mengatur data dalam struktur tabel sehingga proses pencarian, penyimpanan, dan modifikasi menjadi jauh lebih cepat.
  • Skalabilitas: Database mampu menangani data dalam jumlah besar dan mendukung banyak pengguna dalam waktu yang bersamaan.
  • Integritas: Database mendukung aturan validasi, relasi antar tabel, dan mekanisme kontrol untuk menjaga data tetap konsisten dan benar.

Mengapa suatu aplikasi menggunakan database dibandingkan file biasa?

Database mendukung struktur data kompleks dan akses efisien. Walaupun database nantinya lebih mudah digunakan untuk aplikasi dengan skala yang besar dan bisa dijalankan di server (bukan local), alasan utama aplikasi menggunakan database tetap; yaitu agar data yang aplikasi butuhkan dapat disimpan dengan terstruktur untuk memudahkan akses.

Sekilas Tentang Database

Secara luas, database adalah metode penyimpanan data secara terstruktur. Struktur database memungkinkan data untuk diatur dalam format yang mudah dicari, disaring, dan dimanipulasi. Meskipun istilah "database" bisa berarti sekumpulan data yang disusun secara terstruktur, dalam praktiknya istilah ini merujuk pada sistem pengelolaan data terstruktur.

Untuk mengelola database, kita membutuhkan sebuah perangkat lunak bernama DBMS (Database Management System). DBMS adalah sistem yang mengatur proses pembuatan, pembacaan, pengubahan, dan penghapusan data secara efisien. DBMS memungkinkan kita untuk menggunakan database dengan memberikan fitur-fitur pengelolaan database.

Contoh DBMS: MySQL, PostgreSQL, SQLite, MongoDB, dan lainnya.


SQL dan Database Relasi

SQL (Structured Query Language) adalah bahasa standar yang digunakan untuk mengakses dan mengelola database relasional, yaitu database yang menyimpan data dalam bentuk tabel. Setiap tabel terdiri dari baris (data) dan kolom (atribut).

Ciri khas dari database relasional:

  • Data disimpan dalam tabel-tabel yang memiliki relasi satu sama lain
  • Setiap tabel bisa dihubungkan melalui kolom tertentu (misalnya dengan memakai id, user_id)
  • SQL digunakan untuk menulis query seperti SELECT, INSERT, UPDATE, dan DELETE
SELECT * FROM mahasiswa WHERE jurusan = 'Teknologi informasi';

Sederhananya, apa itu SQL?

SQL (Structured Query Language) adalah bahasa pemrograman yang bisa digunakan oleh aplikasi untuk mengakses database relasi. SQL sendiri tidak menjalankan database, database dijalankan melalui aplikasi-aplikasi DBMS. SQL hanyalah bahasa yang digunakan untuk mengakses data. Jadi, DBMS membuat Database berjalan yang setelahnya database ini dapat diatur melalui SQL.

Sekilas Tentang Implementasi Database

Dalam praktiknya, database bisa diimplementasikan dalam dua cara utama:

Local Database (seperti SQLite)

Database langsung disimpan dalam satu file (.db) dengan akses langsung oleh aplikasi, tanpa server. Cocok untuk aplikasi skala kecil atau aplikasi offline. Database lokal juga lebih condong digunakan untuk aplikasi desktop dan bahkan digunakan pada beberapa fitur (seperti pencarian) di sistem operasi.


Client/Server Database (seperti MySQL)

Database dijalankan dalam sebuah server database. Aplikasi (client) mengakses database melalui koneksi jaringan (TCP/IP) untuk mengirimkan instruksi yang nantinya akan diterima oleh server dan diproses oleh server. Database server cocok untuk aplikasi web, sistem multi-user, dan aplikasi skala besar karena bisa diakses dari banyak tempat sekaligus.

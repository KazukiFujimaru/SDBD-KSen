Time Complexity

Struktur Data dapat mempengaruhi waktu pemrosesan dari data yang ada didalamnya. Sehingga diperlukan struktur data yang berbeda dalam penggunaan yang berbeda.


Pengertian Time Complexity

Time complexity (kompleksitas waktu) adalah ukuran yang digunakan untuk menghitung seberapa efisien sebuah algoritma dalam menyelesaikan tugasnya berdasarkan jumlah langkah yang dibutuhkan relatif terhadap ukuran input.

Dengan kata lain, time complexity menunjukkan berapa banyak operasi yang dilakukan oleh algoritma saat ukuran data meningkat. Ini penting karena algoritma yang terlihat sederhana bisa menjadi sangat lambat ketika menangani data dalam jumlah besar.

Time Complexity sendiri akhirnya dapat menjadi patokan untuk melihat seberapa cepat suatu algoritma dapat bekerja.

Contoh:

  • Jika sebuah algoritma butuh 10 langkah untuk memproses 10 data, dan 100 langkah untuk 100 data, maka kompleksitasnya bisa jadi linear.

Hubungan antara Struktur Data dan Time Complexity

Setiap struktur data memiliki karakteristik yang berbeda dalam hal cara menyimpan dan mengakses data. Hal ini mempengaruhi kecepatan dalam melakukan operasi seperti pencarian, penyisipan, dan penghapusan.

Sebagai contoh:

  • Mencari data dalam list biasa (array) membutuhkan pengecekan satu per satu (linear search), dengan time complexity O(n).
  • Sedangkan mencari di dictionary (struktur data hash di Python) bisa dilakukan secara langsung, dengan time complexity O(1).

Jadi, pemilihan struktur data yang tepat dapat mempercepat proses komputasi dalam program kita.

Struktur data hash adalah struktur data yang menggunakan fungsi hash untuk memetakan kunci ke nilai tertentu. Fungsi hash mengubah kunci menjadi indeks dalam sebuah array, sehingga memungkinkan akses data secara langsung (constant time) tanpa perlu melakukan pencarian sekuensial. Di Python, dictionary dan set adalah contoh implementasi dari struktur data hash.


Dimana Time Complexity Berada?

Time complexity muncul dalam banyak proses dalam program, khususnya pada:

  • Akses data (mengambil nilai dari suatu struktur data)
  • Pencarian data (mencari apakah suatu nilai ada dalam struktur data)
  • Penambahan dan penghapusan data

Time complexity berdampak langsung pada waktu proses aplikasi. Jika aplikasi harus memproses data besar dan algoritmanya tidak efisien, maka:

  • Proses akan lebih lambat
  • Pengguna akan merasa aplikasi tidak responsif

Maka dari itu, memahami time complexity sangat penting dalam membuat program yang efisien.


4.Big-O Notation

Big-O notation adalah cara untuk menyatakan time complexity suatu algoritma secara matematis dan umum. Notasi ini menunjukkan pertumbuhan jumlah operasi relatif terhadap jumlah input.

Beberapa contoh Big-O notation:

  • O(1) : Konstanta. Waktu tetap tidak tergantung jumlah data.
    • Contoh: Mengakses elemen ke-i dari sebuah list.
  • O(log n) : Logaritmik. Waktu bertambah lambat meskipun data bertambah besar.
    • Contoh: Binary search pada list yang sudah diurutkan.
  • O(n) : Linear. Waktu bertambah sebanding dengan jumlah data.
    • Contoh: Linear search pada list.
  • O(n^2) : Kuadratik. Waktu bertambah sangat cepat saat data bertambah.
    • Contoh: Nested loop (loop di dalam loop), seperti bubble sort.

Dengan memahami notasi Big-O, kita dapat membandingkan efisiensi algoritma dan struktur data, lalu memilih yang paling tepat untuk kasus tertentu.
